Spawn failed的解决方案

发表于2023-05-25|更新于2023-09-03|hexo
|阅读量:

前言

发布中反复出现的Spawn Failed相当烦人，老是复发，我今天记录一下我目前遇到的问题的解决方案。

解决方案

法一

在博客根目录下设置repo为以下值能解决一些问题

法二

如果还是不行，用命令ssh -T git@github.com检查一下是否连接着

  1. 如果说是没有权限，那就再生成一次公共密匙ssh-keygen -t rsa -C "Your Email"再放在GitHub上的ssh中

  2. 如果失败了并显示could not resolve hostname github.com，那就先在Github IP查询这个网站中查看IP并记录（我写的时候地址是140.82.114.3），再在电脑的一个目录下找到一个hosts文件C:\Windows\System32\drivers\etc，打开Windows自带的搜索，输入记事本，右键以管理员身份运行，打开之后点击文件，打开，选择刚刚说的hosts文件，在最下面一行写入140.82.113.3 github.com并保存，然后再运行ssh -T git@github.com查看一下是否连接成功

  3. You‘ve successfully authenticated, but GitHub does not provide shell access.

    这个也找了半天，最终确定了一种暴力解法：

    1. 删除根目录的.deploy_git
    2. 输入git config --global core.autocrlf false
    3. 最后一键三连hexo cl&hexo g&hexo de
